# Setup.exe v0.5.0 — bundled tray + autostart

Date: 2026-05-23
Status: implementation

## TL;DR

v0.4.x setup.exe ships only the daemon. v0.5.0 ships the daemon AND the new
Windows tray app (`winapp/`, PR #42) in one installer, with a Run-key entry
so the tray icon appears in the system tray on every Windows login —
no terminal, no `python -m …` invocation. Mirrors the Mac DMG that just
shipped, which opens AgentFlow.app in the menu bar after install.

## Goals

1. One download → one install action → user sees AgentFlow icon in tray
   immediately and on every subsequent login.
2. No regression of v0.4.6 daemon install path (scheduled task + auth.json).
3. Bundle size stays under 60 MB (v0.4.6 was ~42 MB).
4. CI on `windows-latest` (Windows-specific build is the explicit exception
   to `feedback_self_hosted_runner` — ff-france is Linux, can't build Win).

## Non-goals (v0.5.0)

- Code signing (separate PR #77).
- Windows named-pipe local IPC for tray ↔ daemon (#94 — tray uses REST only).
- NSIS / Inno-Setup richer wizard (Phase 2 if user demand surfaces).
- Antivirus / SmartScreen warning bypass (documented only).

## Options considered

| Option | Pros | Cons | Pick |
|---|---|---|---|
| A: extend existing PyInstaller spec, ship one setup.exe that bundles daemon + tray, post-install spawns tray as detached process | one download, one install action, parity with Mac DMG | needs two `EXE()` blocks in spec | **YES** |
| B: separate `agentflow-tray-setup.exe` | clean blast radius | two downloads, two install flows, two registry surfaces | no |
| C: NSIS / Inno-Setup wrapper around PyInstaller outputs | richer UX (custom icon, autostart checkbox, uninstaller registry entry) | new toolchain in CI, extra ~2 hours, blocks ship today | Phase 2 |

Pick A for v0.5.0 — minimum-viable single-file install.

## Implementation

### 1. PyInstaller spec — `installer/agentflow-setup.spec`

- Already calls `collect_submodules("agentflow_computer_mcp")` → `winapp.*`
  is auto-included.
- Add `pystray` + `pkgutil-style` discovery for `winapp/assets/*.png`.
- Add a second `EXE()` block named `agentflow-tray` that re-uses the same
  Analysis + PYZ but boots from a tiny `installer/tray_entry.py` script.

### 2. `installer/tray_entry.py` — new

One-liner that imports `agentflow_computer_mcp.winapp.__main__` and calls
`main()`. Same Python path resolution as `setup_gui.py`.

### 3. Post-install autostart — `installer/setup_gui.py`

Extend `_run_install_steps`:

- After existing scheduled-task registration for the daemon, also write
  `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\AgentFlowTray =
  "<LOCALAPPDATA>\AgentFlow\agentflow-tray.exe"`.
- Spawn the tray immediately as a detached process so the icon appears
  in the system tray right away (no logout/login required).
- Wizard adds a checkbox «Запускать иконку в трее при старте Windows»
  default ON. Off → skip the autostart write + skip the immediate spawn.

### 4. Daemon binary copy — `installer/setup_gui.py`

- Existing logic copies `agentflow-desktop.exe` (the setup wizard binary).
- Add a second copy: drop `agentflow-tray.exe` next to it in
  `%LOCALAPPDATA%\AgentFlow\`. Both come from the same release ZIP /
  GitHub asset (see release-installer.yml below).

### 5. Smoke test — `installer/smoke.py`

- Existing `check_daemon_entrypoint_imports` covers daemon.
- Add `check_tray_entrypoint_imports` — `agentflow_computer_mcp.winapp`,
  `winapp.__main__`, `winapp.autostart` import without error.

### 6. Release workflow — `.github/workflows/release-installer.yml`

- Build both EXEs from the spec.
- Run `agentflow-tray.exe --version` as a post-build smoke (no GUI loop;
  just argparse path → exit 0).
- Pack BOTH into the release: `agentflow-desktop-setup.exe`,
  `agentflow-tray.exe`, `SHA256SUMS` covers both.

## Critique / sharp edges

- **Antivirus**: PyInstaller .exes flagged by Windows Defender SmartScreen.
  No fix in v0.5.0 — README + RAG document the «More info → Run anyway»
  workaround. Real fix = code-signing cert (#77).
- **Empty `auth.json` on first launch**: tray must show «Подключи AgentFlow»
  state, NOT crash. `winapp/state.py` already handles missing creds —
  smoke verifies via `tests/winapp/test_tray_menu.py`.
- **Daemon + tray fight over socket file**: tray reads-only via REST, daemon
  writes. Named-pipe IPC = backlog #94. v0.5.0 keeps tray HTTP-only.
- **Stale tray after auto-update**: tray reads `LOCALAPPDATA\AgentFlow\agentflow-tray.exe`.
  Updater replaces both binaries atomically; tray respawns on next boot.
  Documented as known sharp edge.
- **Bundle size**: `pystray` + `Pillow` adds ~4 MB. Budget 60 MB holds.

## Verification

- `python installer/smoke.py` exits 0 — including new tray entry-point check.
- CI on `windows-latest`: build → run `setup.exe --daemon --selftest`
  (existing) AND `agentflow-tray.exe --version` (new). Both exit 0 within
  30s.
- Manual: install on a fresh Win 11 VM, reboot, confirm tray icon visible.
  (Owner does this on their workstation post-merge — out of scope for CI.)
